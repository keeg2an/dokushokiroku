## レビュー観点

### 1. コーディング規約の遵守
CLAUDE.md に定義された規約に沿っているか確認する。
- 命名規則（変数・関数はキャメルケース、定数はアッパースネークケース）
- マジックナンバーが定数化されているか
- 省略変数名が使われていないか（`b`, `e`, `el` 等）
- コメントが「なぜ」を説明しているか（「何を」ではなく）

### 2. エラーハンドリング
- 例外処理は適切な箇所に書かれているか
- CustomException を使うべき箇所で使われているか
- エラーが握り潰されていないか（空の catch ブロック等）

### 3. セキュリティ
- SQLインジェクションの危険がないか
- XSS の危険がないか（ユーザー入力を DOM に直接挿入していないか）
- 機密情報がハードコードされていないか

### 4. パフォーマンス
- N+1 問題が発生していないか
- 不要な DB アクセスや API コールがないか
- ループ内で重い処理を繰り返していないか

### 5. テスタビリティ
- 関数が単一責任になっているか（1関数1役割）
- 副作用が分離されており、ユニットテストを書きやすい設計か
- 依存関係が外部から注入できるようになっているか

---

各観点について「問題なし」「要改善」「改善案」の形式で出力してください。
